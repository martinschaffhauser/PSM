PSM wofür?

Goldstandard für den Vergleich einer Variable ist die randomisiserte kontrollierte Studie (RCT). Kovariaten (Variablen neben der Variable im Hauptinteresse) werden hier durch Zufall auf die 2 (oder mehr) zu vergleichenden Gruppen verteilt und man kann davon ausgehen, dass das einzige wodurch sich die 2 Gruppen unterscheiden die zu untersuchende Variable ist bzw. Kovariaten keinen Einfluss auf die Zuordnung zu einer Gruppe haben (weil eben randomisiert).

In Observationsstudien ist das anders - ihr sammelt Daten aus stattgehabtem klinischem Alltag, wo ihr die Entscheidung zur Behandlung (ja/nein) nicht per Zufall sondern per Indikation trefft. Das können dann einige Kovariaten sein, die zur Behandlungsentscheidung geführt haben - damit unterscheiden sich natürlich dann die Gruppe mit Behandlung und die ohne Behandlung in diesen Variablen voneinander - es liegt also ein systematischer Bias vor.

Hier kommt propensity score matching (PSM) ins Spiel - wobei im Folgenden dann gezeigt wird, dass die Diskussion PSM vs. multiple Regression nicht entschieden ist. Beim PSM wird ein propensity score (PS) aus Variablen errechnet, von denen man weiß bzw. vermutet, dass sie mit der Gruppierungsvariable (zB Behandlung ja/nein) kausal assoziert sind. Der PS ist also ein einzelner Score, der aus mehreren Variablen die Wahrscheinlichkeit der Gruppenzuordnung angibt - also mehrere Dimensionen auf eine einzelnte reduziert.

Nachdem man für den gesamten Observationsdatensatz den PS für alle Patienten berechnet hat, kann man jetzt Patienten (zB behandelte und nicht behandelte) mit gleichen/ähnlichen PS nach verschiedenen Algorithmen miteinander matchen und in Folge die Gruppierungsvariable auf ihren Effekt auf eine Outcomevariable untersuchen; also zB schauen ob die Behandlung einen Effekt auf einen gewählten Outcomeparameter hat.

Datensatzsimulation

Um zu zeigen, wie der Ablauf eines PSM ist und auch die Ähnlichkeit zur multiplen Regression herauszuarbeiten, habe ich folgendes (etwas dümmliches, weil fehlende Kreativität) toy example simuliert. Es wird ein Datensatz generiert, dessen Eigenschaft wir ganz genau kennen, so können wir die Methodik prüfen. Bei einem klinisch erhobenen Datensatz wissen wir die Populationsparameter natürlich nicht.

Kurze Beschreibung vom toy example:

  • Outcome: Blutdruck (BP), kontinuierlich - sagen wir systolisch, wie gesagt: nicht zu ernst nehmen, wird teils recht tief geschätzt 😄

  • Gruppierungsvariable: Blutdruckmedikation (RRMed), binär

  • Geschlecht: 1 = weiblich, binär

  • Alter: Jahre, kontinuierlich

  • Beruf: Chirurg vs Internist vs Statistiker

Für die PS Berechnung wird bauen wir einen bekannten Zusammenhang zwischen den Kovariaten Alter und Geschlecht in den PS für Gruppenzuordnung ein (für das Alter einen quadratischen Zusammenhang: Junge und Alte bekommen eher keine BP Medikation und für Männer eine höhere Wahrscheinlichkeit eine Medikation zu bekommen - alles ein toy example als proof of principle - macht natürlich keinen echten Sinn).

Außerdem simulieren wir einen Effekt von RRMed, Alter, Geschlecht und Beruf auf den Blutdruck (BP):

\[ BP = 80 + 0.5 * Alter + 15 * Geschlecht + 10 * Chirurg + 20 * Statistiker + Internist - 40 * RRMed + error \]

# for random generation
set.seed(2020) 

# 0 = w, 400 w, 600 m
Geschlecht <- rep (0:1, c(400,600)) 
# normalverteiltes Alter, nur positive (abs())
Alter <- round(abs(rnorm(1000, mean=45, sd=15)))  
# another categorical variable
Beruf <- rep(c("Chirurg", "Internist", "Statistiker"), c(300,300,400))

# known PS
# RR-Medikation ja/nein hat in diesem Beispiel eine quadratische Relation mit
# Alter sowie eine höhere Wahrscheinlichkeit für Geschlecht = m\
# (x-μ)/σ is z score https://www.simplypsychology.org/z-score.html
z <- (Alter - 45) / 15 - (Alter - 45) ^ 2 / 100 + 2 * Geschlecht
# plot(z %>% sort())
# https://de.wikipedia.org/wiki/Logit,
# https://search.r-project.org/CRAN/refmans/LaplacesDemon/html/logit.html,
# inverse logit transformation - transform vector to probabilities
### example
# library("SciViews")
# a <- c(1,4,2,6,4,8,2,5,9,4) %>% sort()
# logit <- exp(a) / (1 + exp(a)) # transform vector to probabilities
# ln(logit/(1-logit)) %>% sort() # transform probabilities to vector again
# plot(a)
# plot(logit)
truePS <- exp(z) / (1+exp(z)) 
# plot(tps %>% sort())
# plotting the relationship
# plot(Alter, truePS)
# now we have a vector of propensity scores that are related to Alter and
# Geschlecht --> draw from uniform distribution randomly 1000 times --> if the
# value in the vector is smaller than the value in the truePS vector treatment
# is going to be 1 and the other way round
RRMed <- as.numeric(runif(1000) < truePS)

# creating a data generation process for the outcome measure BP
# https://www.youtube.com/watch?v=fDFyG_q2xQs
# https://stats.oarc.ucla.edu/r/modules/coding-for-categorical-variables-in-regression-models/
error <- rnorm(1000, 0, 5) # error term

BP <- 80 + 
  0.5 * Alter +
  15 * Geschlecht +
  10 * I(Beruf == "Chirurg") +
  20 * I(Beruf == "Statistiker") +
  I(Beruf == "Internist") -
  40 * RRMed +
  error

PS Estimation

Nun soll der propensity score - also die Wahrscheinlichkeit (0 bis 1) aufgrund von gewählten Kovariaten der Gruppe mit Medikation vs der ohne Medikation zugeordnet zu werden - geschätzt werden. Es müssen hier klinisches Wissen bzw. Vermutungen, die eine Assoziation nahelegen, berücksichtigt werden.

Zu Schätzung gibt es verschiedene Methoden. Meistens wird hier eine logistische Regression als Modell verwendet. Man kann das auch über Neural Networks machen, aber das sind dann “Blackboxes” deren Regeln man nicht kennt. Außerdem kenne ich mich damit nicht aus.

# Alter linear
m.out1 <- matchit(RRMed ~ 
                   Alter +
                   Geschlecht, 
                 data = data,
                 distance = 'logit')

# Alter quadratisch
m.out2 <- matchit(RRMed ~ 
                   poly(Alter, 2) + 
                   Geschlecht, 
                 data = data,
                 distance = 'logit')
estimatedPS <- m.out1$distance # Estimated PS
truePS <- data$truePS # Because we only use complete cases
RRMed <- data$RRMed

df <- data.frame(True = truePS, 
                 Estimated = estimatedPS,
                 RRMed = RRMed)

ggplot(df, aes(x = True, y = Estimated, colour = RRMed)) +

              slope = 1, 
              colour = "#990000",
              linetype = "dashed") +
  expand_limits(x = c(0,1),
                y = c(0,1)) +
  ggtitle("Zusammenhang PS und Alter linear modelliert")

Hier wird ersichtlich, dass die PS Schätzung nicht trivial ist. Im ersten Plot entsprechen die geschätzten PS überhaupt nicht den tatsächlichen PS. Erst im zweiten Plot wo der quadratische Zusammenhang aus Alter und RRMed berücksichtigt wird sieht man, dass die geschätzten den echten PS entsprechen.

Wir wissen hier in diesem Beispiel ja genau, wie der PS in der Behandlungsgruppe zu sein, oder eben nicht zustande kommt, können ihn also exakt schätzen. Weiß man das allerdings nicht, so kann man hier natürlich wild daneben liegen und falsche Scores berechnen, womit ein Match dann natürlich nicht so gut funktioniert bzw. der Bias nicht hinaus gearbeitet wird.

Matching

Mit einem Modell, das die PS perfekt schätzt (weil uns die Relationen aus Kovariaten und PS ja bekannt sind) geht es nun zum Match. Ziel ist es die jeweilige Abstände zwischen den PS von Matches zu minimieren und auch für den gesamten Datensatz eine möglichst geringe mittlere Abweichung zwischen den PS zu erreichen. Matches können 1:1 bis 1:n zwischen treatment und control passieren.

Hier gibt es diverse Algorithmen.

Nearest neighbor matching ist zB ein Alogrithmus der sich einfach durch die Liste an Patienten geht (in gegebener Reihenfolge) und sich den jeweils besten Match sucht. Kann natürlich dazu führen, dass insgesamt nicht der beste Match passiert, weil immer nur ein Match pro Iteration beachtet wird (greedy algorithm).

Ein anderer Algorithmus ist genetic matching, den ich mir noch nicht so genau angeschaut habe, der aber gute Ergebnisse bringt.

Parameter, die die Güte des Match beurteilen sind SMD (standardized mean difference) und VR (variance ratio) - SMD sollte so nah wie möglich an 0 sein. Cutoffs aus der Literatur sind zB 0.1 SMD (Stuart, Lee, and Leacy (2013))

Hier ein Beispiel mit nearest neighbor matching. Es werden 1:1 Fall und Kontroll gematcht und pro Iteration jeweils der beste Match, also kleinste Distanz zwischen den PS gewählt. PS wurde hier 100% richtig geschätzt (Alter quadratisch + Geschlecht):

m.out.nn <- matchit(RRMed ~ 
                   poly(Alter, 2) +
                   Geschlecht,
                 data = data,
                 distance="logit",
                 method="nearest",
                 m.order = "largest",
                 caliper = 0.01,
                 replace=FALSE)

Der Effekt von RRMed ist hier sehr ähnlich: 40,74019 wie aus dem gematchten Datensatz mit 358 Patienten (42,5556). Hier lässt sich jetzt diskutieren, welches Ergebnis “richtiger” ist. In der multiplen Regression werden 1000 Patienten verwendet, und tragen sicherlich zu dem etwas anderen Ergebnis bei. Hier ist allerdigs der PS 100% richtig (weil simuliert), was mit einer echten Population nicht möglich ist.

Conclusio

Die Entscheidung für PSM oder multiple Regression sollte man am konkreten Beispiel durchdenken. Es gibt auch ein einige Publikationen die hier die Methoden vergleichen (die ich mir bisher nicht näher angesehen habe). Ziel dieser kleinen Simulationsstudie hier war es, das Problem heraus zu arbeiten und einen kleinen Überblick zu schaffen.

Wir müssen das zusammen dann besprechen - wenn ich den Datensatz kenne, wird es natürlich auch konkreter. PSM hat jedenfalls auch einige Vorteile. Wenn man zB allzu große Regressionsmodelle hat, dann hat man bald mal ein Problem mit den Freiheitsgraden, besonders wenn das Sample nicht so groß ist. Außerdem müssen wir evtl auch das machen, was ansonsten so in der Literatur im fraglichen Bereich gemacht wird (ihr seid ja nehm ich an so irgendwie auf das propensity score matching gekommen).

Na gut, ich freu mich auf eine Zusammenarbeit mit euch.
